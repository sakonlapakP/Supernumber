# เงื่อนไขการ Import CSV เบอร์โทรศัพท์

เอกสารนี้เป็น source of truth สำหรับการเตรียมไฟล์ CSV ที่ใช้ในหน้าแอดมิน `นำเข้าข้อมูลเบอร์ (CSV)`.

## รูปแบบไฟล์

- ไฟล์ต้องเป็นนามสกุล `.csv` เท่านั้น
- Encoding แนะนำเป็น UTF-8 พร้อม BOM เพื่อให้ภาษาไทยอ่านได้ถูกต้อง
- Header ที่แนะนำ:

```csv
เบอร์,ราคาเบอร์,แพ็กเกจ,เครือข่าย,สถานะ
```

ระบบยังรองรับชื่อคอลัมน์ราคาเดิม เช่น `ยอดจ่ายก้อนแรก`, `ยอดชำระแรก`, `ราคา`, `price`, `initial_payment_price` เพื่อ backward compatibility.

## คอลัมน์

| คอลัมน์ | จำเป็น | รายละเอียด |
| --- | --- | --- |
| `เบอร์` | ใช่ | เบอร์ 10 หลัก เช่น `0812345678`; ถ้าไฟล์มี 9 หลัก ระบบจะเติม `0` หน้าให้ |
| `ราคาเบอร์` | รายเดือนจำเป็น, เติมเงินเว้นว่างได้ | ราคาที่ลูกค้าต้องจ่ายสำหรับเบอร์นั้น ตัวเลขเท่านั้น เช่น `12900`; เติมเงินที่ยังไม่มีราคาเว้นว่างได้ |
| `แพ็กเกจ` | ใช่ | เติมเงินให้ว่างหรือใส่ `-`; รายเดือนต้องเป็นรหัสแพ็กเกจ เช่น `TRUE-SV-1499` |
| `เครือข่าย` | ไม่จำเป็น | รองรับ `true`, `true_dtac`, `ais`, `dtac`; ถ้าเว้นว่างจะเป็น `true_dtac` |
| `สถานะ` | ไม่จำเป็น | รองรับ `active`, `hold`, `sold` หรือภาษาไทย `ว่าง`, `จอง`, `ขายแล้ว`; ถ้าเว้นว่างจะเป็น `active` |

## เงื่อนไขเบอร์เติมเงิน

- `แพ็กเกจ` ต้องว่างหรือเป็น `-`
- `ราคาเบอร์` เว้นว่างได้
- ถ้า `ราคาเบอร์` ว่าง ระบบจะบันทึกเบอร์ไว้ในฐานข้อมูล แต่เบอร์นั้นจะยังไม่แสดงเป็นเบอร์พร้อมขายหน้าเว็บ เพราะ catalog ใช้เฉพาะเบอร์ที่มีราคา
- ตัวอย่าง:

```csv
เบอร์,ราคาเบอร์,แพ็กเกจ,เครือข่าย,สถานะ
0812345678,12900,-,true,active
0822345678,,-,ais,active
```

## เงื่อนไขเบอร์รายเดือน

- `แพ็กเกจ` ต้องเป็น `phone_packages.code` ที่มีอยู่จริงในระบบ
- แพ็กเกจต้อง `active`
- `service_type` ของแพ็กเกจต้องเป็น `postpaid`
- `network_code` ของแพ็กเกจต้องตรงกับคอลัมน์ `เครือข่าย`
- `ราคาเบอร์` จำเป็น และใช้เป็นยอดที่ลูกค้าต้องจ่ายสำหรับเบอร์นั้น
- ตัวอย่าง:

```csv
เบอร์,ราคาเบอร์,แพ็กเกจ,เครือข่าย,สถานะ
0912345678,1499,TRUE-SV-1499,true,active
```

## การอัปโหลดและผลกระทบต่อข้อมูลเดิม

- อัปโหลดได้เฉพาะไฟล์เติมเงิน, เฉพาะไฟล์รายเดือน หรืออัปโหลดทั้งสองไฟล์พร้อมกัน
- ระบบจะ retire เฉพาะเบอร์ `active` ของประเภทที่อัปโหลดและไม่อยู่ในไฟล์นั้น โดยเปลี่ยนเป็น `sold`
- ถ้าอัปโหลดเฉพาะเติมเงิน ระบบจะไม่ retire เบอร์รายเดือน
- ถ้าอัปโหลดเฉพาะรายเดือน ระบบจะไม่ retire เบอร์เติมเงิน
- เบอร์ซ้ำในไฟล์เดียวกันจะถูก reject
- เบอร์ซ้ำระหว่างไฟล์เติมเงินและรายเดือนจะถูก reject

## ตัวอย่างไฟล์รายเดือน TRUE

```csv
เบอร์,ราคาเบอร์,แพ็กเกจ,เครือข่าย,สถานะ
0645155461,1199,TRUE-SV-1199,true,active
0645155491,1499,TRUE-SV-1499,true,active
```

```csv
เบอร์,ราคาเบอร์,แพ็กเกจ,เครือข่าย,สถานะ
0961414656,12900,-,true,active
0615154749,,-,ais,active
```
